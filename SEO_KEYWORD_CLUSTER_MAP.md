## Scope

This inventory is built for `FLOW by Crescora AI` using the current public repo plus the stated product vision.

Validation sources used in this checkout:

- `README.md`
- `src/lib/site-content.ts`

This document intentionally separates:

- product-led software/platform terms
- agency/service-provider terms
- available-now capabilities
- near-term / QA capabilities
- vision-only capabilities

No search-volume or keyword-difficulty numbers are included because the repo does not contain verified data from Search Console, Keyword Planner, Ahrefs, Semrush, or equivalent tools.

## Maturity Model

- `Available now`
  - Supported directly by current public copy, route structure, or SEO templates.
  - Examples in repo: WhatsApp automation, web chat, lead capture, booking, reminders, payments, document collection, CRM/API sync, support automation, human handoff, hospitals, clinics, real estate, education, service businesses, analytics/observability themes.
- `In QA / near-term`
  - Present in SEO templates, comparisons, trust/control language, or supporting docs, but not yet broad enough to claim as a mature flagship feature everywhere.
  - Examples in repo: RAG / knowledge assistant, observability dashboard as a stronger product story, diagnostics-specific landing pages, multilingual state SEO, broader orchestration modules.
- `Vision`
  - Mentioned indirectly, implied by legal / acceptable-use scope, or aligned with full product direction, but not strongly supported by current public product pages.
  - Examples in repo: Telegram, SMS, email-first automation positioning, voice QA / conversation intelligence depth, formal cost-control positioning, enterprise governance suites, insurance / finance verticalization, ecommerce verticalization.

## Regulated-Term Flags

Use extra-care language for these themes because the repo policy text explicitly calls out regulated or sensitive workflows:

- healthcare
- finance
- insurance
- KYC / identity verification
- document collection
- education
- emergency or triage flows
- compliance / security claims

Copy guardrails:

- do not imply autonomous medical, financial, eligibility, or legal decisions
- use `assist`, `route`, `collect`, `notify`, `handover`, and `support`
- avoid unqualified claims like `HIPAA compliant`, `medical diagnosis AI`, `automated underwriting`, or `fully autonomous KYC approval` unless verified outside this repo
